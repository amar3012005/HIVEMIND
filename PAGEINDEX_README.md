# PageIndex — Hierarchical Memory Index for HIVEMIND

**Status**: ✅ Implemented (2026-04-07)
**Inspired by**: Visual Copilot's Sitemap + PageIndex Architecture

---

## Overview

PageIndex is an **optional optimization layer** that adds hierarchical memory organization to HIVEMIND. It stores memory IDs in a tree structure (topics/projects) without duplicating content.

### Key Features

| Feature | Description |
|---------|-------------|
| **Hierarchical navigation** | Browse memories by topic/project tree (max 4 levels) |
| **Auto-classification** | LLM classifies new memories during ingestion |
| **Hybrid search** | PageIndex + VectorDB fusion for better recall |
| **Auto-evolution** | CSI agent restructures hierarchy every 6 hours |
| **Graceful fallback** | System works without PageIndex enabled |

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    USER QUERY / NEW MEMORY                   │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  PAGEINDEX (Blueprint Layer) — Hierarchical, IDs Only       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  /projects/davinci/technical → [mem_001, mem_002]   │   │
│  │  /tara/clinical → [mem_101, mem_102, ...]           │   │
│  │  /knowledge/ai-ml → [mem_601, mem_602, ...]         │   │
│  └─────────────────────────────────────────────────────┘   │
│  • Sub-10ms traversal                                        │
│  • Embedding-powered semantic nav                            │
│  • Auto-evolving structure                                   │
└─────────────────────────────────────────────────────────────┘
         │                                    │
         │ (collect memory IDs)               │ (parallel)
         ▼                                    ▼
┌──────────────────────┐           ┌─────────────────────────────┐
│  PostgreSQL          │           │  VectorDB (Qdrant)          │
│  (Lexical Storage)   │           │  (Semantic Search)          │
│  Full memory content │           │  Vector embeddings          │
└──────────────────────┘           └─────────────────────────────┘
         │                                    │
         └────────────────┬───────────────────┘
                          │ (fusion + ranking)
                          ▼
              ┌───────────────────────┐
              │  Full Memories        │
              │  (deduplicated,       │
              │   ranked, enriched)   │
              └───────────────────────┘
```

## Schema

```prisma
model PageIndexNode {
  id          String   @id @default(uuid())
  userId      String   @map("user_id")
  orgId       String?  @map("org_id")
  
  // Hierarchy
  parentId    String?  @map("parent_id")
  parent      PageIndexNode?  @relation("PageIndexHierarchy")
  children    PageIndexNode[] @relation("PageIndexHierarchy")
  
  // Metadata
  label       String   // "Clinical Reasoning", "DaVinci AI"
  path        String   @unique // "/projects/davinci/technical"
  nodeType    String   @default("topic") // root, category, topic, subtopic
  depth       Int      @default(1) // 1-4
  
  // Memory assignment (IDs only)
  memoryIds   String[] @default([]) @map("memory_ids")
  memoryCount Int      @default(0) @map("memory_count")
  
  // Auto-evolution
  autoGenerated Boolean @default(true)
  customBranch  Boolean @default(false) // Protected from pruning
  lastPrunedAt  DateTime?
  
  // Cross-references
  crossRefs   String[] @default([])
  
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  deletedAt   DateTime?
}
```

## Deployment

### Quick Start

```bash
# Run deployment script
/opt/HIVEMIND/scripts/deploy-pageindex.sh
```

### Manual Steps

1. **Run migration**:
   ```bash
   cd /opt/HIVEMIND/core
   npx prisma migrate deploy
   ```

2. **Backfill existing memories**:
   ```bash
   node core/scripts/pageindex-backfill.js --concurrency=10
   ```

3. **Restart server**:
   ```bash
   pm2 restart hivemind-core
   # or
   systemctl restart hivemind-core
   ```

4. **Verify**:
   ```bash
   # Test tree endpoint
   curl http://localhost:8050/api/pageindex/tree
   
   # Test search
   curl -X POST http://localhost:8050/api/search/pageindex \
     -H 'Content-Type: application/json' \
     -d '{"query":"clinical reasoning","limit":10}'
   ```

## API Endpoints

### Get Hierarchy Tree

```
GET /api/pageindex/tree?depth=4&rootPath=/hivemind
```

Response:
```json
{
  "tree": [
    {
      "id": "uuid",
      "label": "Projects",
      "path": "/hivemind/projects",
      "depth": 2,
      "memoryCount": 47,
      "children": [...]
    }
  ]
}
```

### Get Nodes for Memory

```
GET /api/pageindex/memory/:id/nodes
```

Response:
```json
{
  "nodes": [
    {
      "id": "uuid",
      "label": "Clinical Reasoning",
      "path": "/tara/clinical"
    }
  ]

```

```

  "target_node_id": "uuid"
}
```

### Search with PageIndex

```
POST /api/search/pageindex
{
  "query": "clinical reasoning",
  "limit": 20
}
```

## Auto-Evolution

The PageIndex Evolution Agent runs every 6 hours and:

### Pruning (removes sparse nodes)
- Deletes nodes with <5 memories older than 30 days
- Reassigns memories to parent node
- Protected: customBranch=true nodes

### Splitting (creates subtopics)
- Splits nodes with >100 memories
- Uses LLM to detect subtopics
- Creates child nodes and distributes memories

### Merging (combines similar siblings)
- Merges sibling nodes with >70% keyword overlap
- Combines memory IDs
- Deletes redundant node

## Fallback Behavior

| Failure Mode | Fallback | User Impact |
|--------------|----------|-------------|
| PageIndex table missing | Direct hybrid search | None |
| Classification timeout | Async retry | None |
| PageIndex search timeout | VectorDB + PostgreSQL | Slightly slower |
| VectorDB down | PostgreSQL lexical | Degraded recall |
| All layers down | Error message | Search unavailable |

**Guarantee**: Ingestion never fails due to PageIndex. Classification is async and non-blocking.

## Troubleshooting

### PageIndex table not found
```bash
# Run migration
npx prisma migrate deploy
```

### Classification failing
```bash
# Check Groq API key
echo $GROQ_API_KEY

# Test classifier manually
node -e "import('./core/src/services/pageindex-classifier.js').then(m => console.log('OK'))"
```

### Backfill stuck
```bash
# Remove checkpoint and retry
rm .pageindex-backfill-checkpoint.json
node core/scripts/pageindex-backfill.js --concurrency=5
```

### Search returning empty results
```bash
# Check if PageIndex is initialized
curl http://localhost:8050/api/pageindex/tree

# If empty, run backfill
node core/scripts/pageindex-backfill.js
```

## Comparison: With vs Without PageIndex

| Aspect | Without PageIndex | With PageIndex |
|--------|-------------------|----------------|
| **Navigation** | Search only | Browse + search |
| **Topic-scoped queries** | Full vector scan | Sub-10ms node lookup |
| **Memory organization** | Flat (tags only) | Hierarchical tree |
| **Cross-referencing** | Multiple tags | Multiple parent nodes |
| **Auto-evolution** | None | CSI restructures hierarchy |
| **Profile integration** | Separate table | Linked via PageIndex |
